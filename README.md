# AI Token Factory ROI

`aitoken` 是一个面向国产 AI 计算卡智算集群的投资、成本、Tokens 收入和租卡收入测算工具。  
当前仓库是从原项目的 `v3` 版本独立抽出的完整开发包，并已开辟 `v4` 开发版本。默认以寒武纪 `MLU590` 和新华三 H3C 网络方案为基线，可直接继续二次开发。

## 适用场景

- 测算 10,000 到 100,000 卡级别智算集群的建设投资。
- 比较“卖 Tokens”与“对外租卡”两种商业模式的收益差异。
- 评估 GPU、服务器、网络、存储、机柜、水电、OPEX 和折旧对 ROI 的影响。
- 给投资评审、方案设计和商务报价提供可调参数的分析底稿。

## 主要能力

- 按 GPU 卡数自动估算服务器台数、机柜数、Leaf/Spine 数量、管理交换机、光模块、线缆和高性能存储。
- 采用实际工程口径估算推理吞吐，不直接使用理论峰值：
  - 显存带宽约束
  - 算力上限约束
  - 连续批处理增益
  - 模型运行效率
  - 集群利用率
  - 可售卖率
  - 服务可用性
- 支持多模型混合部署，默认包含：
  - Doubao-Seed-2.0-Code
  - DeepSeek-R1-250528
  - GLM-4.7
  - Qwen3-Max
  - Kimi K2.6
- 支持 Tokens 工厂与租卡模式的：
  - 年收入
  - 年 OPEX
  - 折旧
  - 会计利润
  - 累计现金流
  - ROI
  - 回本周期
- 支持市场价格自动采集与用户手动覆盖。
- 支持将完整页面导出为带图表的 PDF 报告。

## 默认方案口径

### 加速卡

- 默认卡型：`寒武纪 MLU590`
- 默认配置：
  - 96 GB HBM
  - 2000 GB/s 显存带宽
  - 315 TFLOPS FP16
  - 550 W TDP

### 网络

- Leaf/Spine：`H3C S9827-128DH`
  - 128 个 400GE QSFP112
  - 102.4 Tbps
  - 典型功耗 850 W
- 管理网：`H3C S5130S-52S-HI`
- 光模块：`H3C QSFP112-400G-VR4-MM850`
- 光纤：`MPO12/APC OM4`

### v3 新增/强化内容

- 400G 光模块数量计算。
- 服务器到 Leaf、Leaf 到 Spine 的光纤链路数量计算。
- 高性能存储可用容量与原始采购容量拆分。
- PDU、机柜内辅材、年度 OPEX 单列。
- H3C 交换机与 400G QSFP112 光模块的采价逻辑。
- 当公开价格样本不足时，自动采价不会贸然覆盖默认价格。

### v4 新增/强化内容

- 为每个模型增加 Tokens 单价模式：
  - `市场价`：同步市场价格时自动更新输入价和输出价。
  - `手动价`：用户手动录入输入价和输出价，同步市场价格时不会覆盖。
- 修改任一模型的输入价或输出价后，会自动切换为 `手动价`。
- 每个模型可单独点击“同步此模型”，恢复为当前市场价。
- 同步市场价格失败时会显示 HTTP 状态或异常信息，方便诊断。

## 核心计算逻辑

### 1. 服务器规模

```text
服务器台数 = ceil(GPU 卡数 / 每台服务器卡数)
```

### 2. 网络规模

```text
Leaf 台数 = ceil(服务器台数 / floor(Leaf 下联端口 / 每服务器网卡端口))
Leaf-Spine 链路 = Leaf 台数 * Leaf 上联端口
Spine 台数 = ceil(Leaf-Spine 链路 / Spine 端口数)
光模块数 = 服务器-Leaf 链路 * 2 + Leaf-Spine 链路 * 2
光纤链路数 = 服务器-Leaf 链路 + Leaf-Spine 链路
```

### 3. 机房功耗

```text
IT 功率 = GPU 功率 + 服务器非 GPU 功率 + 网络功率 + 存储功率
设施功率 = IT 功率 * PUE
年用电量 = 设施功率 * 8760
```

### 4. 年度 OPEX

```text
年度 OPEX = 年电费 + 年水费 + 机柜租赁 + 运维费 + 软件平台费 + 公网/安全费用
```

### 5. 推理吞吐

```text
单卡工程上限 TPS =
  min(
    显存带宽 * 带宽效率 / 活跃权重 GB * 连续批处理增益,
    FP16 算力 * 500 * 算力效率 / 活跃参数 B
  )
  * 副本适配率
  * 模型运行系数

单卡实际 TPS =
  单卡工程上限 TPS
  * 服务利用率
  * 可售卖率
```

### 6. Tokens 收入

```text
Tokens 净收入 =
  (输入 Tokens / 1e6 * 输入单价 + 输出 Tokens / 1e6 * 输出单价)
  * 官方价成交系数
  * (1 - 收入扣减率)
```

### 7. ROI

```text
累计现金 ROI =
  (-CAPEX + Σ(净收入 - OPEX)) / CAPEX

年度会计利润 =
  净收入 - OPEX - 折旧
```

### 健康检查

```text
GET /api/health
```

### 获取默认市场参数

```text
GET /api/market/defaults
```

### 刷新市场价格

```text
GET /api/market/refresh
```

刷新接口会尝试采集：

- 火山引擎 / 扣子模型费用
- 阿里云百炼模型规格与计费
- Moonshot / Kimi API 价格
- H3C S9827-128DH 公开报价与招采样本
- 400G QSFP112 光模块公开价格

如果某些页面抓取失败，接口会保留默认值并返回 `warnings`。

## Tokens 单价设置

模型价格表中的输入价和输出价单位都是：

```text
元 / 百万 tokens
```

每个模型都有独立的价格模式：

- `市场价`：默认模式，点击“同步市场价格”时会跟随后端采集结果更新。
- `手动价`：适合录入合同价、渠道价、折扣价或内部测算价；同步市场价格不会覆盖。

如果想把某个模型从手动价恢复为市场价，可以点击该行的“同步此模型”。

## 后续开发建议

- 若要支持更多国产卡型，可先扩展 `accelerators`，再增加不同卡型的默认功耗、价格和吞吐参数。
- 若要支持更多模型，可扩展 `models` 并补充官方价格来源。
- 若要做更真实的生产测算，可继续细化：
  - 网络收敛比
  - 多机房 / 多 POD
  - 备件池
  - 服务器冗余
  - 液冷与 CDU
  - 人员编制
  - 税费、融资成本和现金回收周期
- 若要做正式投委会版本，建议引入情景分析：
  - 乐观 / 基准 / 保守
  - 卡价波动
  - 电价波动
  - Tokens 单价下行
  - 利用率爬坡

- 本仓库是 `v3` 的独立开发包，不再依赖原始多版本仓库。
